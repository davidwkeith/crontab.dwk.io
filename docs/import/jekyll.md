# Importing from Jekyll

Jekyll is the original static site generator (51k GitHub stars), tightly integrated with GitHub Pages. It uses Markdown with YAML frontmatter and Liquid template tags.

See [ssg-migrations.md](ssg-migrations.md) for shared SSG migration guidance including frontmatter conventions, template syntax stripping patterns, image handling, and redirect generation. This doc covers only what's specific to Jekyll.

## How it detects this platform

Config file: `_config.yml` in the project root, combined with either a `Gemfile` containing `jekyll` or a `_posts/` directory.

Disambiguate from Hexo: if `_config.yml` exists AND `package.json` contains `"hexo"`, it's Hexo, not Jekyll.

## Where content lives

- **Blog posts**: `_posts/` with date-prefixed filenames: `YYYY-MM-DD-slug.md`
- **Pages**: root directory or `_pages/` (if configured in `_config.yml`)
- **Drafts**: `_drafts/` (no date prefix, unpublished)
- **Images**: `assets/images/`, `images/`, or `assets/img/` (varies by theme)
- **Collections**: `_collection_name/` (custom content types)

### Filename convention

Jekyll posts MUST be named `YYYY-MM-DD-title.md`. The date in the filename is the publish date. Extract the date and slug from the filename:

```
2024-03-15-my-great-post.md
‚Üí publishDate: "2024-03-15"
‚Üí slug: "my-great-post"
```

## Frontmatter mapping

| Jekyll field | Anglesite field | Notes |
| --- | --- | --- |
| `title` | `title` | Direct copy |
| `description` or `excerpt` | `description` | Jekyll may auto-generate excerpts from first paragraph |
| `date` | `publishDate` | Often redundant with filename date; prefer frontmatter if present |
| `tags` | `tags` | Direct copy (array or space-separated) |
| `categories` | `tags` | Merge with tags |
| `image` or `header.image` | `image` | Varies by theme |
| `published` | `draft` | Invert: `published: false` ‚Üí `draft: true` |
| `permalink` | ‚Äî | Custom URL pattern ‚Äî use for redirect mapping |
| `layout` | ‚Äî | Skip (Anglesite uses its own layouts) |

Jekyll's `tags` can be either an array (`tags: [a, b]`) or space-separated (`tags: a b`). Handle both.

## Content conversion

Jekyll content is Markdown with Liquid template tags:

**Liquid tags to strip:**
- `{% highlight lang %}...{% endhighlight %}` ‚Üí standard code fences
- `{% include file.html %}` ‚Üí remove (include content is layout-specific)
- `{% link path/to/page.md %}` ‚Üí resolve to the page URL
- `{% post_url YYYY-MM-DD-slug %}` ‚Üí resolve to `/blog/slug`
- `{{ site.baseurl }}` ‚Üí remove (Anglesite handles base URLs)
- `{{ page.title }}` ‚Üí replace with the actual title value
- `{% if %}...{% endif %}` ‚Üí remove conditional blocks (keep the content if simple)
- `{% for %}...{% endfor %}` ‚Üí remove loop blocks

**Kramdown syntax** (Jekyll's default Markdown renderer):
- `{:.class-name}` attribute syntax ‚Üí strip
- `{: #id}` ID syntax ‚Üí strip
- `{:toc}` table of contents ‚Üí strip (Anglesite can generate its own)

## Image handling

Jekyll images are typically in `assets/images/` or `images/`. References in content use:
- `{{ site.baseurl }}/assets/images/photo.jpg`
- `/assets/images/photo.jpg`
- `![alt]({{ "/assets/images/photo.jpg" | relative_url }})`

Strip Liquid filters (`| relative_url`, `| absolute_url`) and resolve the path. Copy image files to `public/images/blog/`.

## URL patterns for redirects

Jekyll's default permalink is `/:categories/:year/:month/:day/:title/`. Custom permalinks are set in `_config.yml` under `permalink:`.

Common patterns:

| Permalink setting | Example URL |
| --- | --- |
| `date` (default) | `/category/2024/03/15/my-post/` |
| `pretty` | `/category/my-post/` |
| `/:title/` | `/my-post/` |
| `none` | `/category/2024/03/15/my-post.html` |

Read `_config.yml` to determine the permalink structure.

## Common issues

- **Date in filename vs frontmatter**: Both exist. If they conflict, prefer the frontmatter `date` field.
- **Collections**: Jekyll `_collections/` are custom content types (portfolio items, testimonials). Ask the owner how to handle these ‚Äî import as posts or create page stubs.
- **GitHub Pages plugins**: Some Jekyll sites use plugins (`jekyll-feed`, `jekyll-seo-tag`, `jekyll-sitemap`) that add metadata. These don't affect content but may explain missing frontmatter fields (e.g., descriptions auto-generated by `jekyll-seo-tag`).
- **Sass/CSS in `_sass/`**: Ignore during content import, but note if the owner asks about visual design.
- **Data files**: Jekyll's `_data/` directory contains YAML/JSON/CSV files used in templates. These don't import directly but may contain navigation, team member lists, or other structured data worth reviewing.
